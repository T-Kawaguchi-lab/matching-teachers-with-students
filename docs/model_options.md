# モデル構成と理由

## デフォルトで入れている構成
1. **multilingual-e5-large**
   - 利点: 日本語・英語混在でも意味的な近さを取りやすい。TRIOSの研究課題、論文、修論テーマのような自由記述に強い。
   - 欠点: 同じ専攻内で似たテーマが多いと、全員のスコアが高止まりしやすい。

2. **TF-IDF 文字 n-gram**
   - 利点: 研究テーマ内の専門語、固有名詞、略語の違いを拾いやすい。
   - 欠点: 意味的には近いが表現が違うテーマは取りこぼしやすい。

3. **研究分野ラベルの exact match 加点**
   - 利点: 「最適化」「時系列解析」など同じラベルが出たときに直感に合う加点ができる。
   - 欠点: ラベル設計が粗いと差がつきにくい。

4. **同一専攻内キャリブレーション（行方向 z-score）**
   - 利点: 同じ専攻で全体的に似ている場合でも、その学生にとって相対的に誰がより近いかを見やすくできる。
   - 欠点: 絶対値の意味より、相対順位の意味が強くなる。

5. **MMR による委員会多様化**
   - 利点: 主指導と副指導2名が全員ほぼ同じタイプになるのを防ぎやすい。
   - 欠点: 単純な上位3名とは一致しない場合がある。

## 今回、別モデルも使ったほうがよいか
### 結論
**はい。E5単独より、E5 + TF-IDF + exact match + MMR のハイブリッドが向いています。**

同じ専攻の中で「みんな似ている」問題に対しては、意味ベクトルだけより、
- 用語の一致
- ラベルの一致
- 相対順位の補正
- 委員会の多様化

を合わせるほうが差が出ます。

## 追加候補（今回は説明だけ。必要なら後で導入可能）
### BGE reranker / cross-encoder 系
- 利点: 上位候補10名程度を再精査するときに強い。似たテーマの細かい順序が改善しやすい。
- 欠点: 重い。Cloud Run の起動時間・メモリ消費が増える。全組み合わせに使うと遅い。
- 推奨: **上位10名だけ再順位付け**するオプションとして導入。

### LLM による研究分野抽出
- 利点: 「研究分野候補」の自然さが上がる。
- 欠点: APIコスト、再現性、出力ゆらぎがある。
- 推奨: 最初は未導入。必要なら管理画面から再生成できる形で追加。
